# <a name="getevents"></a>GetEvents

L’élément **GetEvents** représente l’opération utilisée par les clients de l’extraction pour demander des notifications à partir du serveur. 
  
[GetEvents](getevents.md)
  
```xml
<GetEvents>
   <SubscriptionId/>
   <Watermark/>
</GetEvents>
```

 **GetEventsType**
## <a name="attributes-and-elements"></a>Attributs et éléments

Les sections suivantes décrivent les attributs, éléments enfants et éléments parents.
  
### <a name="attributes"></a>Attributs

Aucun.
  
### <a name="child-elements"></a>Éléments enfants

|**Élément**|**Description**|
|:-----|:-----|
|[SubscriptionId (GetEvents)](subscriptionid-getevents.md) <br/> |Représente l’identificateur d’un abonnement est interrogé pour les événements.  <br/> |
|[Filigrane](watermark.md) <br/> |Représente le dernier filigrane retourné au client. Si GetEvents n’a pas été appelée pour cet abonnement, le client utilise le filigrane renvoyé à partir de la demande Subscribe. Dans le cas contraire, la limite à partir du dernier événement de la dernière réponse GetEvents est utilisée.  <br/> |
   
### <a name="parent-elements"></a>Éléments parents

Aucun.
  
## <a name="remarks"></a>Remarques

Le schéma qui décrit cet élément se trouve dans le répertoire virtuel EWS de l'ordinateur qui exécute Microsoft Exchange Server 2007 sur lequel le rôle de serveur d'accès au client est installé.
